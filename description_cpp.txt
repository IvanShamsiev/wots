
Тестовое задание: Игра World of Tinyships.

В игре уже присутствует простейшая логика авианосца и вся необходимая графика. Требуется реализовать логику работы самолётов, а именно - простейшую физику полёта и искусственный интеллект самолёта, подчиняющийся игроку.
Основные требования:
- на борту корабля должно быть 5 самолётов
	- будучи на борту самолёты никак не отрисовываются в мире
- по правому клику мыши в воздух поднимается очередной самолёт
	- самолёт должен начать отрисовываться в мире
	- самолёт должен начать взлетать вдоль палубы корабля, медленно набирая скорость
- самолёт при движении должен учитываться следующие параметры
	- время полёта (настроечный параметр)
	- максимальная скорость (настроечный параметр)
	- максимальная угловая скорость (настроечный параметр)
- по истечении времени полёта самолёт должен вернуться на корабль
	- самолёт должен долететь до своего корабля, только после этого он считается севшим
	- после посадки самолёт больше не рисуется в мире
	- после посадки самолёт может быть снова поднят в воздух только по истечении определённого времени, требующемся для заправки самолёта (настроечный параметр)
- по левому клику мыши назначается новая цель для самолётов
	- самолёты, находящиеся в воздухе, летят в направлении цели и там летают вокруг неё на некотором расстоянии

Управление в игре:
- WASD и стрелки управляют движением корабля
- правый клик мыши поднимает очередной самолёт в воздух
- левый клик мыши назначает цель для самолётов
- пробел перезапускает игру

Что будет оцениваться:
- физичность полёта самолётов с точки зрения обычного игрока
- выполнение требований к поведению самолётов
- читаемость и аккуратность кода

Пояснения по структуре проекта:
- файлы проекта для Visual Studio 2017 находятся в папке project_vs2017
- файлы проекта для Visual Studio 2015 находятся в папке project_vs2015
- файлы проекта для Code::Blocks находятся в папке project_codeblocks
- код-заготовка этой игры находится в файле game.cpp. Здесь должна происходить основная работа.
	- на данный момент настроечные параметры находятся в namespace params
	- логика работы корабля находится в классе Ship. Сейчас она реализована только на базовом уровне, необходимом для тестирования игры, но не для самой игры, и сильно страдает от недостатка физичности. Доработка поведения корабля не входит в тестовое задание.
- в остальных файлах находится минимально необходимый "фреймворк" для обеспечения работы игры. Эти файлы не предполагаются для изменения при выполнении тестового задания.
- при необходимости в проект можно добавлять новые файлы
- код игры должен использовать только предоставленный "фреймворк" (scene::что-то и game::что-то) и стандартную библиотеку C++ (std::что-то). Никакие другие библиотеки не должны использоваться для выполнения тестового задания.

Краткая справка по доступным для кода игры функциям "фреймворка":
- scene::createShipMesh - создаёт новую модель корабля
- scene::createAircraftMesh - создаёт новую модель самолёта
- scene::destroyMesh - удаляет модель, созданную любой из двух предыдущих функций
- scene::placeMesh - распологает модель в мире в указанных координатах и повёрнутой под нужным углом

Стадии полёта самолёта:
- Самолёт должен взлетать с палубы, как бы она не поворачивалась.
Полёт начинается, когда взлёт совершён
- Взлёт должен быть постепенным, в зависимости от параметра LINEAR_SPEED
- После взлёта самолёт продолжает разгон, но в это же время начинается поворот
в сторону цели
- Цель может быть изменена в процессе полёта и поворотов, если цель находится в другой стороне, 
самолёт должен плавно начать поворачивать в другую сторону
- Дальше идёт полёт по прямой до тех пор, пока самолёт не долетит до места, 
либо не изменится цель
- Если новая цель возле самолёта и угловой скорости достаточно, чтобы вращаться вокруг неё, 
следует перейти на вращение вокруг неё
- Если новая цель возле самолёта и угловой скорости не достаточно, 
следует отдалиться от неё на достаточное расстояние и начать вращение
- Дальше самолёт кружит вокруг цели на некотором расстоянии, пока не кончится время полёта, 
либо не изменится цель (тогда начинаем снова с разворота к цели)
- Когда время полёта истекло, самолёт начинает двигаться в сторону корабля

По остаточному принципу:
- Реализовать посадку вдоль палубы
(возможно, заблокировать движение корабля на этот период)
- Реализовать поочерёдный взлёт самолётов с задержкой на взлёт предыдущего
